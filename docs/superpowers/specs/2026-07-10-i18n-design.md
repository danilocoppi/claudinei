# i18n (en / es / pt-BR) — Design

**Data:** 2026-07-10
**Status:** Aprovado (brainstorming) → pronto para plano de implementação

## Objetivo

Internacionalizar a UI do Claudinei com **react-i18next** (decisão do usuário —
opção B sobre a alternativa de mini-i18n próprio): idiomas **en (primário/
default)**, **es** e **pt-BR**, com um **seletor de idioma com bandeirinhas à
direita do logo** na sidebar. A escolha persiste em `localStorage`. Troca
instantânea (re-render), sem reload.

## Decisões

1. **Biblioteca:** `i18next` + `react-i18next` (padrão da indústria). SEM
   `i18next-browser-languagedetector` — o default é **sempre `en`** na
   primeira visita (pedido explícito: en primário), não o idioma do browser.
   Persistência manual: chave `claudinei.locale` no `localStorage`.
2. **Chaves tipadas:** declaração `i18next.d.ts` com `resources: typeof en`
   (declaration merging oficial do react-i18next) — chave inexistente vira
   erro de compilação.
3. **Escopo: frontend apenas.** Mensagens de erro vindas do backend (ex.:
   "projeto tem uma sessão ativa") são exibidas como chegam. i18n do backend
   fica fora (projeto separado, se um dia).
4. **Suíte de testes existente** asserta textos em pt-BR (~120 asserções): o
   setup de teste (`web/src/test/setup.ts`) inicializa o i18n com
   `lng: 'pt-BR'`. Um teste dedicado cobre o default `en` (localStorage
   vazio). Nenhuma asserção existente é reescrita.
5. **`<html lang>`** acompanha o idioma (init + `languageChanged`).
6. **Fallback:** `fallbackLng: 'en'` — chave sem tradução em es/pt-BR cai
   para o inglês (nunca mostra a chave crua).

## Seletor de idioma (frontend-design, Glass/Aurora)

```
│ ✳ Claudinei        [🇺🇸] │   ← pill compacto à direita do logo (bandeira atual)
│        (clique ▾)         │
│      ┌───────────────┐    │
│      │ 🇺🇸 English  ✓ │    │   ← menu glass flutuante (PORTAL — lição do
│      │ 🇪🇸 Español    │    │     backdrop-filter da sidebar)
│      │ 🇧🇷 Português  │    │
│      └───────────────┘    │
```

- Botão pill mostra **só a bandeira** do idioma atual (`title` com o nome);
  borda glass, hover eleva.
- Menu: bandeira + **nome nativo** (English / Español / Português), ✓ no
  ativo, hover `glass-bg-strong`. Fecha com clique fora e `Esc`.
- Componente: `web/src/components/LanguageSwitcher.tsx`, renderizado na linha
  do logo na `Sidebar.tsx` (`display:flex; justify-content:space-between`).

## Estrutura

```
web/src/i18n/
├── index.ts      init do i18next (resources, lng do localStorage ?? 'en',
│                 fallbackLng 'en', persistência em languageChanged, html lang)
├── en.ts         dicionário base (fonte da tipagem)
├── es.ts         : typeof en (tipado — faltar chave = erro)
├── pt-BR.ts      : typeof en
└── i18next.d.ts  declaration merging p/ chaves tipadas
```

- Convenção de chaves por área: `sidebar.*`, `status.*`, `dashboard.*`,
  `chat.*`, `terminal.*`, `modal.*`, `confirm.*`, `mural.*`, `tasks.*`,
  `notify.*`, `common.*` (Cancelar/Salvar/Enviar…).
- Interpolação padrão do i18next: `t('confirm.deleteTitle', { name })`.
- `main.tsx` importa `./i18n` antes do render.

## O que é traduzido (tudo do frontend)

- `Sidebar` (Terminais, + Terminal, sem sessão, Interação entre terminais,
  Mural, Tarefas, títulos das ações Reviver/Iniciar/Editar/Excluir, empty state)
- `STATUS_LABEL` → chaves `status.*` (o Record em `types.ts` é substituído
  por uso de `t()`; testes que o importavam passam a asserir o texto pt-BR)
- `Dashboard`, `ProjectCard`, `NewProjectModal` (criar/editar), `StartSessionModal`,
  `ConfirmDialog`, `FolderPicker`, `EmojiPicker`, `ColorField`
- `ChatView` (Abrir no terminal, aviso in_terminal, turno concluído/erro),
  `ChatInput` (placeholders, avisos de upload), `MessageBlock` (Pensamento,
  Encaminhar, encaminhado para {name}, subagente, comandos), `ToolCallCard`
- `TerminalView` (Encerrar terminal, desconectado/reconectando, Reconectar)
- `MuralPanel`, `TasksPanel` (títulos, colunas, estados)
- `notifications.ts` (fora do React — usa `i18n.t` direto, ponto forte da lib)
- Título do documento permanece "Claudinei" (marca, não traduz)

## Tratamento de erros / bordas

| Situação | Comportamento |
|---|---|
| localStorage indisponível/corrompido | cai para `en` silenciosamente |
| Chave sem tradução em es/pt-BR | fallback `en` |
| Mensagem de erro do backend | exibida como chegou (fora do escopo) |
| Idioma trocado com modais abertos | re-render instantâneo (estado do i18next) |

## Testes

- Infra: `t()` resolve nas 3 línguas; interpolação; fallback en; persistência
  (`localStorage` gravado ao trocar); default `en` sem localStorage.
- `LanguageSwitcher`: abre menu, troca idioma (asserta um texto conhecido da
  sidebar mudando de língua), ✓ no ativo, fecha com Esc/clique fora.
- Suíte existente permanece verde com `lng: 'pt-BR'` no setup.

## Fora de escopo (YAGNI)

- i18n do backend (mensagens de erro da API).
- Detecção do idioma do navegador (default é en, sempre).
- Pluralização complexa/ICU (as strings atuais não precisam).
- Lazy-load de catálogos (3 dicionários pequenos, bundle único).
